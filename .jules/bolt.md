## 2026-05-27 - Optimized Middleware Order and Catch-all Routing
**Learning:** In Express applications, the order of middleware is critical for performance. Placing `express.static` below `logger`, `bodyParser`, and `cookieParser` causes every static asset request (images, JS, CSS) to undergo unnecessary processing, including disk I/O for logging and CPU for parsing headers/bodies that are irrelevant for static files. In this codebase, moving `express.static` above these middlewares reduced static asset response times by ~15% and SPA entry response times by ~9%. Additionally, a catch-all route using `req.url` with `path.parse` can be fooled by query parameters, and missing common extensions like `.woff2` in the exclusion list can cause 404 assets to unnecessarily trigger expensive SPA template rendering.

**Action:** Always place `express.static` as high as possible in the middleware stack, usually right after the favicon. When implementing SPA catch-all routes, use `req.path` for extension checks and ensure a comprehensive list of static asset extensions is excluded to prevent redundant rendering of the main index for missing assets.
